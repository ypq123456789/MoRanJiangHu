# 同人模式提示词总线与境界模板化改造

## 目标
- 将“同人融合”从开局阶段的临时输入提升为存档级正式状态。
- 为世界观生成、开局生成、主剧情、剧情规划、女主规划、世界演变、变量校准建立统一的同人提示词装配链路。
- 把散落在多份提示词中的境界硬编码抽成统一母板，在同人模式下支持“原著优先，不足时回退现体系”。
- 将“世界观生成”和“同人境界体系生成”拆成两次独立请求，分别写入 `core_world` 与 `core_realm`。

## 已确认决策
- 同人默认策略：平行线贴原著。
- 境界默认策略：原著优先，不足则回退现体系。
- 继续复用现有 `OpeningConfig` 与新建游戏“同人融合”输入，不新增 UI 配置项。
- 保留 `角色.境界层级` 作为底层数值承载，只切换提示词口径与 `角色.境界` 文案规则。
- 同人境界体系不再内嵌进 `world_prompt`，而是写入独立的境界体系槽位，避免世界观母本与数值母板职责混杂。

## 为什么这样做
- 当前同人信息只在世界观生成和开局生成阶段可见，进入主回合、世界演变、规划分析和读档后会丢失，导致同一存档前后口径断裂。
- 现有提示词入口分散，直接复制多套“同人版”会快速失控，后续维护会反复漏改。
- 境界表被硬编码在多份提示词中，任何一次调整都要多点同步；抽成母板后才能稳定支持同人体系替换与默认体系回退。
- world_prompt 负责世界母本，realm_prompt 负责数值口径；拆成两次请求后，境界体系可以完整覆盖映射表、阶段推进表、大境突破表和硬边界，而不把世界观写成境界百科。
- 先落盘计划文件能让后续改动、回归和任务交接都基于同一份明确说明执行，降低跨回合协作成本。

## 影响链路
- 运行时状态：`useGameState`、`useGame`、会话生命周期、快速重开。
- 存档：手动存档、自动存档、读档恢复。
- 提示词装配：世界观生成、开局生成、主剧情系统提示词、酒馆模式拼装。
- 独立境界体系生成：realm prompt 生成、校验、回填与后续读取优先级。
- 独立链路：统一规划分析、世界演变、变量校准。
- 数值口径：角色属性、修炼体系、功法体系、NPC 参考、开局 COT、主剧情 COT。

## 实施顺序
1. 新增本计划文件，作为本次任务的交接与验收依据。
2. 将 `openingConfig` 提升为存档级状态并贯通新建、重开、存读档。
3. 新增同人运行时构建器，统一输出同人设定摘要、阶段补丁与境界母板。
4. 将世界观生成与境界体系生成拆成两次请求，并把结果分别写入 `core_world` 与 `core_realm`。
5. 将同人补丁接入开局生成、主剧情 COT、剧情规划、女主规划、世界演变与变量校准。
6. 抽离默认境界母板并为同人模式提供术语替换/体系回退逻辑，同时让独立 realm prompt 覆盖映射、阶段推进、大境突破、终点文案与硬边界。
7. 补充 fandom 压测用例并执行构建验证。

## 验收项
- 非同人存档不注入同人摘要，但会稳定读取默认独立境界体系提示词。
- 同人存档在世界观、开局、主剧情、规划分析、世界演变中都能读取同人设定摘要。
- 同人存档的 `core_world` 与 `core_realm` 分别生成，realm prompt 必须独立覆盖映射表、阶段推进表、大境突破表、终点文案和硬边界。
- 同人档读档后仍保持同人口径，不依赖最近开局缓存。
- `保留原著角色` 与 `融合强度` 会影响实际提示词约束，而不是只影响展示文案。
- 原著有明确体系时使用原著术语；原著无明确体系时回退默认体系，现有数值逻辑不崩。
- `npm run build` 通过；新增 fandom 压测覆盖开局、主剧情、规划分析、世界演变的装配结果。

## 兼容性说明
- 旧存档没有 `openingConfig` 时按 `undefined` 处理，等价于非同人模式。
- v1 不重写底层战斗、经验、修炼数值系统，只在提示词层和状态持久化层扩展。
- 若原著体系缺失或表达不完整，只允许切换术语和边界描述，不改底层数值字段含义。
