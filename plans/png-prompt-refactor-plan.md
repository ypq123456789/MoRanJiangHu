# PNG 提示词重构计划（无旧存档兼容）

## Summary
- 重构 PNG 导入、生图装配、画师串预设消费逻辑：本地剥离 `artist`，单独保存；剩余正面提示词交给 AI 清洗固定 IP / 角色 / 发型 / 外貌 / 构图等内容；负面提示词与 PNG 参数继续完整保留，参数仍按现有规则净化后保存。
- 不做旧存档兼容与懒迁移。直接升级当前数据结构、导入导出结构与运行时装配逻辑；旧格式 JSON / 旧本地数据不保证可读。
- 内部统一为 `前置正向提示词 / 主体正向提示词 / 后置正向提示词 / 负面提示词` 抽象。所有后端共用这套装配模型，NovelAI 最终映射为单一正向 prompt + 独立 negative prompt。
- NAI 资料依据：
  - 官方确认 prompt 顺序会影响结果：[Prompting Basics](https://docs.novelai.net/en/image/strengthening-weaknessening/)
  - 官方 V4/V4.5 使用 `base prompt + character prompts` 组织方式：[Multi-Character Prompting](https://docs.novelai.net/en/image/multiplecharacters/)
  - 官方支持顺序插入块：[Prompt Chunks](https://docs.novelai.net/en/image/promptchunks/)

## Key Changes
- 数据结构：
  - `PNG画风预设结构` 强制保存：`原始正面提示词`、`剥离后正面提示词`、`AI提炼正面提示词`、`画师串`、`画师命中项`、`负面提示词`、`参数`、`原始元数据`。
  - `画师串预设结构` 拆成两段：`画师串` 与 `正面提示词`；其中 `正面提示词` 明确表示“非 artist 风格正面词”。
  - 新增运行时装配结果类型：`前置正向提示词`、`主体正向提示词`、`后置正向提示词`、`最终正向提示词`、`最终负向提示词`。
- 本地 artist 剥离：
  - 新增本地解析器，支持顶层逗号切分，保留 `::权重::`、`() [] {} <>`、转义字符与 token 原顺序。
  - 采用“规则 + 词库”识别 artist：显式模式优先，其次本地 UTF-8 artist 词库匹配。
  - 所有命中的 artist token 原样保存，包含完整权重包装，不改写、不重排、不降权。
- PNG 导入与 AI 提炼：
  - PNG 解析后先本地剥离 artist，再将“剥离后正面提示词”交给 AI 做风格清洗。
  - AI 仅处理非 artist 正面词，不接触负面词和参数。
  - AI 失败时回退为“剥离后正面提示词”，但字段仍完整保存。
- 装配逻辑：
  - 非兼容模式：直接把 `画师串 + 非 artist 正面提示词` 全部放入 `前置正向提示词`；主体词组只负责人物/场景主体；负面词直接合并。
  - 兼容模式：只把 `画师串` 放入 `前置正向提示词`；`非 artist 正面提示词` 作为风格上下文交给 AI，吸收到 `主体正向提示词`；后处理阶段不再重复硬拼这部分正面词；负面词直接合并。
  - `后置正向提示词` 统一承载构图、比例、镜头、环境补强和当前增强词组。
  - NovelAI 最终映射：`prompt = 前置 + 主体 + 后置`；`negative_prompt = 合并后的负面提示词`；`v4_prompt.caption.base_caption` 与最终正向 prompt 保持一致。
- 兼容模式提示词优化：
  - 不向 AI 暴露“兼容模式”术语。
  - 改为直接行为要求，例如：
    - “你必须吸收额外提供的风格正面提示词，并将其自然整理进最终词组中。”
    - “额外提供的画师标签会在最终提示词中单独保留，因此不要重复输出画师名。”
    - “你只需要处理风格、质感、媒介、质量、光照等非主体信息，不要重复主体外观。”
  - 非兼容模式也改为结果导向约束，不再出现“本次已单独追加”之类实现描述。

## Test Plan
- 本地 artist 解析：
  - 普通 artist tag、权重 artist tag、混合 style/artist、重复 artist、无 artist、转义逗号、LoRA 与 artist 并存。
- PNG 导入：
  - NovelAI PNG、SD PNG、无元数据 PNG；确认 `原始正面 / 剥离后 / AI结果 / 画师串 / 负面 / 参数` 全部正确。
  - AI 失败回退时，仍能生成完整预设。
- 装配验证：
  - NPC、场景、秘档三条链路分别覆盖兼容模式与非兼容模式。
  - 非兼容模式验证：前置正向包含 `画师串 + 正面提示词`。
  - 兼容模式验证：前置正向只包含 `画师串`，正面提示词只进 AI，不在后处理重复拼接。
  - NovelAI、OpenAI 兼容后端、SD/ComfyUI 后端都验证最终 prompt 顺序一致。
- 提示词验证：
  - 检查所有兼容模式相关系统提示词与任务提示词中，不再出现“兼容模式”字样。
  - 检查非兼容模式相关提示词中，也不再出现“本次已单独追加”这类产品实现描述。

## Assumptions
- 不保留旧存档兼容，不做自动迁移，不做旧字段回填。
- 词库采用仓库内静态 UTF-8 文件维护。
- PNG 参数继续净化保存，不保留分辨率与 seed。
- 手工画师串预设不经过 AI 重写；只有 PNG 导入会新增“本地剥离 + AI 清洗”。
- 实施直接覆盖现有 PNG / 画师串 / 装配逻辑，不保留旧路径。
