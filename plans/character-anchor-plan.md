## 目标
- 为 NPC 新增“角色锚点”，通过 API 从角色文本资料中提取稳定外观提示词。
- 生图时可启用角色锚点，将稳定外观提示词直接附加到最终 prompt。
- 词组转化器在锚定模式下只生成镜头、动作、构图、表情、氛围、环境等非恒定内容。
- 场景生图若涉及已锚定角色，直接拼接其锚点提示词，而不是重新自然语言描述角色。
- 为角色锚点提供独立的管理界面、手动编辑能力与启用控制。
- 对“词组转化器预设管理”进行结构性重构，使一个预设可以存放多套针对不同生成模式的提示词模板。

## 核心原则
- 锚点是“稳定视觉资产”，不是一次性的风格词。
- 结构化字段用于管理和编辑，真正下发时优先使用完整正面提示词。
- 提取阶段优先保留确定信息，不允许臆造角色没有明确描述的视觉特征。
- 分词器在锚定模式下禁止改写基础外观。
- 场景生图中只对“实际参与该场景的角色”注入锚点。
- 角色锚点与 PNG 画风预设是两套系统，职责不同，不能混用。

## 范围

### 包含
- NPC 角色锚点的数据结构、保存、读取、编辑、删除。
- NPC 文本资料到角色锚点提示词的提取流程。
- 图片管理或社交界面中的角色锚点管理入口。
- NPC 生图工作流接入角色锚点。
- 场景生图工作流接入角色锚点。
- 词组转化器预设结构升级，支持锚定模式专用提示词。

### 不包含
- 直接通过图像反推角色锚点。
- 自动合并多个 NPC 为一个共享锚点。
- 多角色冲突解决的高级 UI 编排器。
- 服装锚点、道具锚点、场景锚点等更细分资产系统。

- 现有“单一提示词”结构不够用了，建议扩展为模板集合：
  - `普通模式提示词`

  - 便于导出和未来做统一预设管理。

## 工作流设计

### 阶段 A：角色锚点提取
- 输入：
  - NPC 姓名
  - NPC 已有描述文本
  - 社交档案/剧情中可用的人物描写
- 输出：
  - 完整正面提示词
  - 可选负面提示词
  - 结构化视觉特征
- 提取规则：
  - 只提取稳定视觉外观。
  - 不提取动作、姿势、镜头、构图、临时情绪。
  - 不提取当前场景背景。
  - 对胸部、身材、发色、瞳色等尽量结构化。
  - 信息不足时留空，不补猜测。

### 阶段 B：NPC 生图时的 prompt 装配
- 锚定关闭：
  - 维持现有逻辑。
- 锚定开启：
  - 角色锚点正面提示词
  - 角色锚点负面提示词
  - 可选服装/状态补充
  - 分词器只产出镜头、动作、构图、表情、环境补充
- 关键约束：
  - 分词器不得重复角色基础外观。
  - 分词器不得覆盖锚点核心描述。

### 阶段 C：场景生图 prompt 装配
- 识别参与场景的 NPC。
- 对存在锚点且允许自动注入的 NPC 拼接锚点。
- 分词器只负责：
  - 场景构图
  - 人物互动
  - 动作分布
  - 镜头与光影
- 需要补的控制：
  - 注入顺序
  - 多角色锚点分隔方式
  - 是否限制场景里最大自动注入角色数

### 阶段 D：手动管理
- 可手动编辑锚点提示词。
- 可从结构化字段回填文本。
- 可重新提取。
- 可禁用自动注入。

## 提示词策略

### 1. 锚点提取提示词
- 目标：
  - 从 NPC 文本中提取稳定视觉特征并输出英文 tag。
- 强约束：
  - 不输出动作、姿势、镜头、构图、场景。
  - 不输出故事性质描述，例如“冷酷”“强大”，除非它能直接视觉化。
  - 不输出不确定特征。
  - 尽量保留全部稳定外观，不做摘要化压缩。

### 2. 锚定模式分词器提示词
- 目标：
  - 只生成镜头、动作、姿态、表情、构图、氛围、环境补充。
- 强约束：
  - 已提供角色锚点，不得重写角色基础外观。
  - 不得覆盖发色、瞳色、体型、胸围等稳定特征。
  - 可以补充临时服装与道具，但必须避免冲突。

### 3. 场景锚定模式提示词
- 目标：
  - 只输出场景镜头和角色互动，不重写已知锚点角色的外观。
- 强约束：
  - 对每个角色仅补其在当前场景的动作与位置关系。

## 风险与难点
- 多角色场景中锚点 prompt 过长，可能影响生成质量。
- 不同模型对胸部、体型、年龄感标签的兼容性差异较大。
- 如果锚点文本过强，可能压制动作和构图词。
- 结构化字段与完整提示词可能发生人工编辑后的不一致。
- 场景角色识别若不准确，会误注入无关锚点。
