# 贡献指南

感谢你愿意参与 `墨色江湖：无尽武林`。

本项目当前仍在快速迭代阶段，欢迎提交 Bug 修复、文档改进、重构建议、功能提案和测试补充。为了让协作过程更顺畅，请先阅读本指南。

## 开始之前

- 请先查看 `README.md`、`docs/` 与 `plans/`，确认你的想法没有和现有规划重复。
- 对于较大的功能改动、架构调整或提示词体系重组，建议先开 Issue 或 Discussion 对齐方向。
- 小型修复、文档更新、明显的类型或构建问题，可以直接提交 Pull Request。

## 开发约定

- 所有新增或修改文件请统一使用 `UTF-8` 编码。
- 请尽量保持改动聚焦，不要顺手重排大量无关代码。
- 如果你修改的是提示词、流程编排或数据模型，请同时检查相关联的运行时模块是否需要同步更新。
- 如果你调整 UI 或交互，建议在 PR 中附上截图、录屏或简短说明。
- 如果你的变更引入了新的环境变量、设置项或操作步骤，请同步更新 `README.md` 或相关文档。

## 推荐流程

1. Fork 仓库并创建一个清晰的分支，例如 `fix/world-sync-error` 或 `docs/readme-refresh`。
2. 安装依赖并在本地运行项目：

```bash
npm install
npm run dev
```

3. 在提交前至少执行与你改动相关的检查：

```bash
npm run build
npm run stress:test
```

4. 提交 Pull Request，并清楚说明背景、改动范围、验证方式和潜在风险。

## Pull Request 建议

请尽量在 PR 描述中回答这些问题：

- 这个改动解决了什么问题
- 为什么采用当前实现
- 是否影响现有存档、设置、提示词或同步数据格式
- 是否需要额外的迁移、部署或配置
- 你如何验证了这次修改

## 提交内容建议

- Commit 信息不强制某一种规范，但请尽量简洁、明确。
- 涉及多类改动时，优先拆分为多个逻辑清晰的提交。
- 不要提交密钥、Token、真实私有数据或本地导出的存档内容。

## 我们特别欢迎

- 文档整理与注释补充
- 构建、类型和运行时稳定性修复
- 提示词链路可读性改进
- 云同步、存档与数据恢复相关测试
- 移动端和桌面端体验细节优化

## 沟通方式

- 普通 Bug、改进建议、使用问题：请优先通过 Issue 提交。
- 涉及安全问题：请不要公开披露，参考 [SECURITY.md](./SECURITY.md)。

感谢你的投入。每一次清晰、克制且可验证的改动，都会让这个项目更适合长期协作。
